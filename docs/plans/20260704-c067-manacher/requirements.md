## 背景

字符串大类已有 3 页，全是「在文本里找模式」的**模式匹配**：KMP（C-062，部分匹配表）、Rabin-Karp（C-063，滚动哈希）、Boyer-Moore（C-064，坏字符跳），共用 KmpView 匹配轨。

**Manacher** 是字符串算法的另一维度——**单串自身的回文结构分析**，求最长回文子串。朴素做法枚举中心向两边扩展是 O(n²)；Manacher 通过两个技巧做到 **O(n)**：① 在字符间插入分隔符 `#` 统一奇偶回文；② 维护「当前最右回文」的中心 C 和右边界 R，对新中心 i **利用回文的对称性**，用镜像点 `2C-i` 的已知半径 `min(R-i, p[2C-i])` 免费初始化，只扩展超出 R 的部分。这是「用已算结果加速」的漂亮范例，与 KMP 的部分匹配表异曲同工却面向回文。

## 目标

在字符串大类新增第 4 页「Manacher 最长回文子串」，接入算法播放器（`AlgorithmPlayer`）：

1. **新建 ManacherView 回文轨**（第 13 条播放器轨，additive 可插拔）：
   - 上行**转换串**（`#` 分隔的字符格）；
   - 下行**半径数组 `p[]`**（每个中心的回文半径，未算显示空）；
   - 高亮：当前中心 i（琥珀环）、镜像点 `2C-i`（蓝环）、当前最右回文带 `[L,R]`（浅蓝底）、目前最长回文（绿底）。
2. **预处理 + 镜像复用 + 中心扩展逐步重走**：`init` 展示转换串 → 每个中心 `mirror`（在 R 内，用对称性初始化后再扩展）或 `expand`（超出 R，从 0 纯扩展）→ `done`（`max(p)` 对应最长回文，标绿）。
3. 固定输入 `"babad"` → 转换串 `#b#a#b#a#d#`、`p=[0,1,0,3,0,3,0,1,0,1,0]`、最长回文 **"bab"**（长 3）；过程含镜像复用（i=5 复用镜像半径 1 后扩展到 3）与边界截断（i=7 被 `R-i` 限制）两种关键情形。

## 验收标准

- `/docs/manacher` 新页：介绍正文（讲清插 `#` 统一奇偶 + 对称性复用把 O(n²) 降到 O(n)）+ 播放器同屏，右侧四语言代码随步高亮。
- 动画：转换串 + 半径数组两行，当前中心/镜像/最右回文带/最长回文分色高亮；走到 `done` 时最长回文 "bab" 标绿。
- 菜单 + 首页「字符串」新增第 4 项（紧接 Boyer-Moore），新图标 `manacher.svg`。
- 全门禁绿（format/lint/type-check/单测覆盖率/e2e）；真机自检确认转换串、半径数组、镜像复用、最长回文 "bab"。

## 非目标

- 不做「求所有回文子串 / 回文子串计数」——本页聚焦「最长回文子串」。
- 不做回文树（PAM）/ 回文自动机——留待后续进阶。
- 不改 AlgorithmPlayer 的既有轨；ManacherView 为新增 additive 轨，其它算法不设 `manacher` 字段即不渲染。
