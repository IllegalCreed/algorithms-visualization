## 背景

数学与数论大类已有两页筛法（埃氏筛 / 线性筛），都用数字网格 SieveView。本页换一个全新视角——**欧几里得算法（辗转相除求最大公约数 GCD）**，用**几何矩形铺砖**来诠释，是最优美的算法可视化之一。

**欧几里得算法**：`gcd(a, b) = gcd(b, a mod b)`，反复取模直到余数为 0，此时的被除数就是最大公约数，O(log min(a,b))。**几何解释**：`gcd(a,b)` 恰好等于能把 `a×b` 矩形**无缝铺满的最大正方形边长**。做法是反复从矩形的长边切下**最大的正方形**（一次能切 `⌊a/b⌋` 个 `b×b` 的），剩下 `(a mod b)×b` 的小矩形，再对它递归；切到某个正方形正好铺满、没有剩余，那个**最小正方形的边长就是 gcd**。这直观解释了取模递推为何成立——切正方形 = 减去若干个 b。

## 目标

在「数学与数论」大类新增第 3 页「欧几里得算法」，接入算法播放器（`AlgorithmPlayer`），**新建第 17 条 GcdView 矩形铺砖轨**：

1. **矩形铺砖**：渲染 `a×b` 矩形（缩放适配画布）；每个除法步从长边切下 `⌊a/b⌋` 个正方形（按步着色、标注边长），本步方块琥珀高亮，剩余子矩形虚线框标出。
2. **逐步递归**：`init`（原矩形）→ 每个除法步 `cut`（切正方形、更新剩余）→ `done`（铺满、gcd = 最小正方形边长）。
3. 固定 `gcd(30, 18)`：除法步 `30=1·18+12`、`18=1·12+6`、`12=2·6+0`；切下 18/12/6/6 四个正方形恰好铺满 30×18；**gcd = 6**（最小正方形边长）。

## 非目标

- 不做扩展欧几里得（求 Bézout 系数 / 模逆元）——本页专注 gcd 本身与其几何直觉，扩展欧几里得可作后续页。
- 不做辗转相减法（subtraction 版）——用标准辗转相除（division 版），铺砖一次切 `⌊a/b⌋` 个正方形。
- 不改 AlgorithmPlayer 既有轨；仅 additive 加 `Step.gcd?` + 一行 `v-if`（其它算法不设 → GcdView 不渲染，零回归）。
