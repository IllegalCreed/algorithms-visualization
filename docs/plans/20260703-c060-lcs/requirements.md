## 背景

动态规划大类已有编辑距离（[[C-20260703-053]]，序列对齐 min）、0-1 背包（[[C-20260703-054]]，优化选择 max），二者都在 MatrixView 上**填表求一个数值**。但 DP 表填完后，**如何从表里恢复出「答案本身」**（不只是答案的大小）——回溯（traceback）——还没有一页演示。

本变更做**最长公共子序列（LCS）**：求两个串的最长公共子序列。它先像前两页一样填一张二维 DP 表求出长度，然后**从右下角沿表回走、恢复出 LCS 字符串本身**。这补齐 DP 的另一半：填表求最优值 + 回溯求最优解。

## 目标

1. 新增 **LCS** 算法页（`/docs/lcs`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. **复用 MatrixView 矩阵轨**（第 4 消费者）+ 极小扩展 `pathCells`（回溯路径高亮），演示「填表 → 回溯恢复解」完整两阶段。
3. 逐步呈现：**填表**（字符相同取左上 +1、不同取上/左较大）→ 表满右下角 = LCS 长度 → **回溯**（从右下角沿匹配走对角、否则走上/左，收集匹配字符）→ 恢复出 LCS 串。
4. 在「动态规划」大类下新增第 3 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到 DP **不只给出最优值、还能回溯恢复最优解**：LCS 表填完后，从右下角一路回走拼出 LCS 字符串。
- 作为学习者，我想看清 LCS 递推与编辑距离的差别：**字符相同 → 左上 +1（对角）**、**不同 → 取上/左较大**。
- 作为学习者，我想对照编辑距离/背包/LCS，理解二维 DP 填表是同一套范式，而回溯恢复解是它的延伸。

## 验收标准

- [ ] `/docs/lcs` 正常渲染：介绍正文 + AlgorithmPlayer（矩阵轨 + 代码区）。
- [ ] 矩阵轨渲染 `(m+1)×(n+1)` DP 表（X=`ABCD` 行、Y=`ACDF` 列 → 5×5），逐格填：相同取左上 +1、不同取 max(上,左)。
- [ ] 填表阶段：`match`（相同，源=左上）/`mismatch`（不同，源=上、左）高亮清晰；`fillDone` 右下角 = LCS 长度（3）。
- [ ] 回溯阶段：从右下角沿路径回走，`pathCells` 绿环高亮整条回溯路径；匹配格收集字符；末步 `done` 恢复出 LCS = `ACD`。
- [ ] 菜单「动态规划」下出现「最长公共子序列」，点击进入正确路由；首页网格同步出现。
- [ ] MatrixView 仅 additive 扩展 `pathCells`（编辑距离/背包/Floyd 不设 → 无 `.mx-path`，零回归）；既有轨 + 全部算法页零回归。
- [ ] 门禁全绿（format/lint/type-check/单测含覆盖率/e2e），真机自检通过。

## 非目标

- 不做「所有 LCS」枚举（本页恢复一条 LCS；正文可带过存在多条）。
- 不做可变输入（两串固定，与既有 DP 页范式一致）。
- 不做空间优化（滚动数组）版本（正文可带过；本页保留完整表以便回溯）。
