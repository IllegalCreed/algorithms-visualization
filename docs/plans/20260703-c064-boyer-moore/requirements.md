## 背景

字符串大类已有 KMP（[[C-20260703-062]]，部分匹配表）+ Rabin-Karp（[[C-20260703-063]]，滚动哈希）。本变更做 **Boyer-Moore**，凑齐字符串匹配「三巨头」。它的独特之处：**从右往左比**，失配时用**坏字符规则**把模式**大步右跳**——若失配的文本字符不在模式里，直接跳过整段（一步跳 m）；在模式里则对齐到它最右出现处。实践中常常「跳着走」、亚线性，是最快的通用串匹配之一。

## 目标

1. 新增 **Boyer-Moore** 算法页（`/docs/boyer-moore`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. **复用 KmpView 字符串匹配轨**（第 3 消费者）+ 极小扩展 `matchedFrom`（已匹配**后缀**高亮，BM 从右往左匹配后缀）。零新轨。
3. 逐步呈现：每个对齐窗口**从右往左**逐字符比；失配 → 按**坏字符表**计算右移量（大步跳）；全匹配 → 命中。坏字符表在变量面板展示。
4. 在「字符串」大类下新增第 3 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到 Boyer-Moore **从模式末尾往左比**（与直觉相反），理解它为何能「跳着走」。
- 作为学习者，我想看清**坏字符规则**：失配字符不在模式里就跳过整段、在模式里就对齐到最右出现处，从而一次跳很多格。
- 作为学习者，我想对照 KMP（部分匹配表）/ Rabin-Karp（滚动哈希）/ Boyer-Moore（坏字符），把串匹配三思路凑齐。

## 非目标

- 不做「好后缀规则」（Boyer-Moore 的第二条规则，较复杂）——本页聚焦更直观的**坏字符规则**；好后缀在正文带过。
- 不做可变输入（文本/模式固定）。
- 不做 Manacher / AC 自动机——留作字符串大类后续。
