## 背景

「回溯与搜索」大类已有 N 皇后（[[C-20260703-055]]，棋盘约束回溯）+ 子集生成（[[C-20260703-056]]，选/不选二叉决策树）。子集页新建的 **DecisionTreeView 决策树轨**是回溯家族的通用原语，本变更做**全排列（Permutations）**——**复用 DecisionTreeView，零轨改动**，验证决策树原语对回溯家族的可复用力（如 GraphView 服务 6 图算法、MatrixView 服务 3 DP）。

全排列与子集是决策树的两种形态对照：子集 = 对每个元素「选 / 不选」的**二叉**决策树；全排列 = 每个位置**从剩余未用元素挑一个**的**多叉**决策树（分支数逐层收窄 3→2→1）。

## 目标

1. 新增**全排列**算法页（`/docs/permutations`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. **复用 DecisionTreeView 决策树轨**（零改动）：仅在 types.ts 加 `PermuteExecPoint` 执行点，无新 View、无 Step 新字段、AlgorithmPlayer 零改动。
3. 逐步呈现全排列的决策树 DFS：**从剩余元素挑一个 → 递归 → 挑下一个 / 回溯**，到叶子（元素全用完）记录一个排列，直到枚举出全部 `n!` 个排列。
4. 在「回溯与搜索」大类下新增第 3 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到全排列在决策树上「每步从没用过的元素里挑一个」的过程，理解排列与子集在决策树形态上的差别（多叉 vs 二叉）。
- 作为学习者，我想看到分支数逐层收窄（3→2→1）如何导出 `n!` 个排列，建立排列数与决策树的对应。
- 作为学习者，我想复用同一套决策树可视化，对照子集/排列，体会「回溯 = 决策树 DFS」是统一框架。

## 非目标

- 不做「组合 / 组合总和 / 括号生成」等其它回溯题（本页只做全排列；决策树轨为它们继续铺路）。
- 不做可变输入（元素集合固定 `[1,2,3]`，与既有回溯页固定输入范式一致）。
- 不做「交换法（swap）」实现变体（正文可带过；本页决策树采用「used 标记 + 挑未用」形式，与决策树「从剩余挑一个」最贴合）。
- 不新建可视化轨（本变更核心即复用 DecisionTreeView）。
