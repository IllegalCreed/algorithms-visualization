## 背景

动态规划大类已有编辑距离（[[C-20260703-053]]）、0-1 背包（[[C-20260703-054]]）、最长公共子序列（[[C-20260703-060]]），三页都是**二维 DP 表**（MatrixView）。但 DP 还有一大类是**一维 dp 数组**——`dp[i]` 只依赖前面的 `dp[j]`。这一形态还没有一页演示。

本变更做**最长递增子序列（LIS）**的 O(n²) DP：`dp[i]` = 以第 `i` 个元素结尾的最长递增子序列长度，`dp[i] = 1 + max(dp[j])`（对所有 `j<i` 且 `a[j]<a[i]`）。它是一维 DP 的经典，补齐 DP 的另一常见形态，并与 LCS 同属「子序列」问题。

## 目标

1. 新增 **LIS** 算法页（`/docs/lis`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. **复用 MatrixView 矩阵轨**（第 5 消费者）做「值 + dp」两行小表：一维 dp 数组横排逐格填，复用 `pathCells` 高亮最终 LIS 位置。零新轨。
3. 逐步呈现：对每个 `i`，**回看所有 `j<i`**，若 `a[j]<a[i]` 且能变长就 `dp[i]=dp[j]+1`；dp 填完后 **max(dp) = LIS 长度**，并高亮构成 LIS 的元素。
4. 在「动态规划」大类下新增第 4 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到**一维 DP** 的样子：`dp[i]` 逐格填、每格回看前面所有 `dp[j]`，区别于前几页的二维表。
- 作为学习者，我想看清「以 i 结尾的最长递增子序列」如何由前面某个 `dp[j]+1` 得来，并最终恢复出具体的 LIS。
- 作为学习者，我想对照 LCS（二维）与 LIS（一维），理解 DP 的不同形态。
