## 背景

栈/队列/数组/链表/树/堆六个结构页已用知识页骨架做完。本变更是 M3 数据结构动画的**第七个**，做**哈希表 Hash**——它换了个找元素的思路：前面的结构都靠「位置（下标）」或「指针」定位，哈希表**靠算**——把键丢进散列函数算出桶下标、直接跳过去，平均 `O(1)`。

具体做**拉链法**：散列 `hash(key) = key % 7`（7 个桶）。两个不同键可能算出同一个桶（**哈希冲突**），拉链法给每个桶挂一条**链**，冲突的键追加到链尾。

## 两个地基决策（brainstorming + 交互原型已确认）

继承 C-015 页种决策，哈希新增两条：

1. **桶阵列 + 拉链可视化**。7 个桶竖排（下标徽章 0–6），每个桶挂一条**横向链**（节点盒序列）。散列把键映射到桶（命中桶徽章高亮主题绿）；冲突的键追加到该桶链尾——**冲突就是看得见的「某个桶的链变长」**。容量限 16，初始预填 `[15,8,23,4]` → 1 号桶 `[15][8]` **一上来就含一个冲突**示例。
2. **散列直达 + 扫链 + 输入框选值**。插入/查找时先 `hash(key)=key%7` 算桶（状态解说算式）、直达该桶；插入空桶直接放、非空追加链尾；查找则**沿桶内链逐个比较**（找到/没找到、解说扫了几次）。值由**输入框（1–99）**给出。

## 哈希与前面结构的差异（一句话）

哈希表用**散列函数算下标**取代「下标定位 / 指针遍历」——`useHash` 暴露 `hash`（key%N）/ `insert`（算桶 + 空放/冲突追加）/ `search`（算桶 + 扫链）/ `has`，桶为 `[id,key][][]` 二维（桶数组 × 每桶链）；可视化从「一条结构」变为「桶阵列 + 每桶一链」，动画核心是**散列直达 + 冲突追加 + 扫链**。

## 业务规则 / 约束

- **互动模型**：读者驱动、组件局部状态；插入/查找数据同步可观察、散列命中 + 扫链为延时高亮动画（setTimeout、卸载清理、busy 防重入、重置可中断）。
- **哈希语义**：`hash(key)=key % 7`；`insert` 算桶 → 查重（已存在不插）→ 空桶直放 / 非空追加链尾；`search` 算桶 → 沿链逐个比较返回 found+steps；`has` 查键在其桶内；容量限 16（满禁插入）；值 1–99（非法提示）；初始 `[15,8,23,4]`（桶1 含冲突）。键唯一（哈希集合语义，不放重复键）。
