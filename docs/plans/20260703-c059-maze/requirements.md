## 背景

「回溯与搜索」大类已有 N 皇后（[[C-20260703-055]]，棋盘约束）、子集/排列/组合总和（[[C-20260703-056]]/[[C-20260703-057]]/[[C-20260703-058]]，决策树线：二叉/多叉/剪枝）。回溯的「棋盘约束」与「决策树枚举」两条线已成体系，但还缺**网格搜索**这一常见形态——在一张地图上找路径。

本变更做**迷宫寻路（老鼠走迷宫）**：在带墙的网格里，用 DFS 从起点找到终点。老鼠沿一个方向前进，撞到死路就退回换方向——这是回溯「前进-受阻-回退」最直观的具象。它给回溯大类补上网格搜索视角，与决策树线呼应。

## 目标

1. 新增**迷宫寻路**算法页（`/docs/maze`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. 新建第 11 条可视化轨 **MazeView 迷宫轨**（墙/通路 + 起点/终点 + 当前格 + 当前路径 + 已访问 + 解路径），借鉴 BoardView 的 CSS grid 布局范式，既有 10 轨零改动。
3. 逐步呈现网格 DFS 与回溯：**沿某方向走一步 → 撞死路 → 退回上一格换方向 → 直到到达终点**，途经的死路清晰可见。
4. 在「回溯与搜索」大类下新增第 5 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到 DFS 在迷宫里**前进、撞死路、回退换方向**的完整过程，把「回溯」理解成走迷宫时的「此路不通就往回走」。
- 作为学习者，我想看清哪些格被走过又放弃（死路）、哪些格构成当前尝试的路径、最终解路径是哪条。
- 作为学习者，我想对照 N 皇后（棋盘）、决策树（子集/排列/组合总和）、迷宫（网格），体会回溯是套在不同「状态空间」上的同一套 DFS。

## 验收标准

- [ ] `/docs/maze` 正常渲染：介绍正文 + AlgorithmPlayer（迷宫轨 + 代码区）。
- [ ] 迷宫轨渲染 5×5 网格：墙（暗块）、起点、终点（🚩）、当前格（🐭）、当前路径（琥珀）、已访问（浅蓝）。
- [ ] 分步演示：`move`（前进一格）、`deadend`（死路）、`backtrack`（退回上一格）、`goal`（到达终点）；至少 2 处死路 + 回溯清晰可见。
- [ ] 末步 `done`：解路径（start→goal 连续、不穿墙）整条标绿（`solved`）。
- [ ] 菜单「回溯与搜索」下出现「迷宫寻路」，点击进入正确路由；首页网格同步出现。
- [ ] 既有 10 轨（Bars/Aux/Stack/Tree/Count/Bucket/Graph/Matrix/Board/DecisionTree）+ 全部算法页零回归；新轨 additive 可插拔（`Step.maze?`）。
- [ ] 门禁全绿（format/lint/type-check/单测含覆盖率/e2e），真机自检通过。

## 非目标

- 不做 BFS 最短路 / A\* 寻路（本页专注 DFS 回溯；正文可带过 BFS 找最短）。
- 不做可变/随机迷宫（迷宫固定，与既有回溯页固定输入范式一致）。
- 不做「所有路径」枚举（本页找到一条路径即停，聚焦回溯过程）。
