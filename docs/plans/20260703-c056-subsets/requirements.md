## 背景

「回溯与搜索」大类首发 N 皇后（[[C-20260703-055]]）用 **BoardView 棋盘轨**演示「棋盘约束回溯」。回溯的本质是**深度优先地搜索一棵决策树**——N 皇后页正文已点出这一点，但棋盘轨并未把「决策树」本身画出来。

本变更做**子集生成（Subsets / 幂集）**，把回溯的另一半心智模型——**决策树**——直接可视化：对每个元素做「选 / 不选」两个分支，DFS 走遍这棵二叉决策树，每到一个叶子就得到一个子集，共 2^n 个。它与 N 皇后配对，形成回溯的两种互补视角：**棋盘约束 vs 决策树搜索**。

## 目标

1. 新增**子集生成**算法页（`/docs/subsets`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. 新建第 10 条可视化轨 **DecisionTreeView 决策树轨**（通用原语：定位节点 + 决策边 + 当前节点/递归路径/已访问/解叶高亮），为后续回溯题（排列、组合、组合总和）复用铺路。
3. 逐步呈现回溯在决策树上的 DFS：**选一个元素 → 递归 → 回溯换「不选」→ 递归**，到叶子记录一个子集，直到枚举出全部 2^n 个子集。
4. 在「回溯与搜索」大类下新增第 2 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想看到回溯**在一棵决策树上前进与回退**的过程，理解「每一步做一个选择、错了/走完就退回换另一个选择」的通用套路。
- 作为学习者，我想看到「选 / 不选」两个分支如何生成 2^n 个子集，建立子集/幂集与二叉决策树的对应。
- 作为学习者，我想对照 N 皇后（棋盘约束）与子集（决策树），理解回溯的两种可视化视角其实是同一套 DFS + 剪枝/回溯。

## 非目标

- 不做「组合 / 排列 / 组合总和 / 括号生成」等其它回溯题（本页只做子集；决策树轨为它们铺路，后续变更承接）。
- 不做可变输入（元素集合固定 `[1,2,3]`，与 N 皇后固定 4×4 同为「定输入细粒度重走」范式）。
- 不做子集的「按选下标 ≥ start」增量枚举变体（本页采用最经典的「逐元素选/不选」二叉决策树形式，决策树最规整）。
