## 背景

- **M8 拍板**（2026-06-30 Owner）：算法页 = **介绍正文 + 交互可视化 + 多语言代码播放器**（三件套）。M8① 已由 6 个新排序（C-040 桶 ~ C-045 鸡尾酒）立起全模板样板。
- **遗留不一致**：M3 时期的 **9 个老排序页仍是「裸播放器」**（`<AlgorithmPlayer :module="xxx"/>` 一行、8 行文件、无正文）：冒泡 / 选择 / 插入 / 希尔 / 归并 / 快速 / 堆 / 计数 / 基数。点进去只有可视化+代码，缺「这是什么、怎么做、复杂度、和谁对照」的讲解——与新排序页体验割裂。
- **本变更（M8③）**：给这 9 页**补齐介绍正文**，升级为全模板，与新排序页对齐。**纯追加正文，不改算法/播放器/轨道/交互**。

## 要做什么

对 9 个裸排序页，逐页：

1. **视图补正文**（`src/views/Article/SortAlgorithm/<Name>.vue`）：裸 `<AlgorithmPlayer>` → `<Article>`（h1 + sub + 2~3 段算法讲解，正文中段嵌入 `<AlgorithmPlayer>`，末尾 Callout 对照/关联）。正文算法特定、引用该页固定演示数据。
2. **spec 加 Case**（`<Name>.spec.ts`）：新增 `TC-VIEW-*-03`——断言含 `Article`、h1 文本含中文标题；保留既有 -01/-02（播放器 + 柱数 + 轨道 + counter）不动。
3. **e2e 扩断言**（`e2e/<name>-sort.e2e.ts`）：加 `expect(page.locator('.article h1')).toContainText('…')`；既有断言（柱数/拖末步升序/轨道）保持。

9 页 = 冒泡[7,6,5,10,9,8,4,3,2,1] / 选择[同] / 插入[同] / 希尔[同] / 归并[同,Aux 轨] / 快速[同,Stack 轨] / 堆[同,Tree 轨] / 计数[3,1,4,1,6,2,3,6,4,1,Count 轨] / 基数[42,7,25,63,18,31,56,9,Count 轨]。

## 不做什么（边界）

- **不改任何算法模块 / 播放器 / 轨道组件 / 类型**：零逻辑改动，纯 `.vue` 正文 + spec + e2e。
- **不改路由 / 菜单 / 首页 / 图标 / TC-HOOK 计数**：9 页早已在菜单里，非新页。
- **不动 6 个新排序页**（已全模板）、不动数据结构/图算法页。
- **不为省事套模板**：每页正文按各自算法特性单独写（冒泡的故事 ≠ 堆的故事），保证质量。
