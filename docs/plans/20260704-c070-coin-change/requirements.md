## 背景

动态规划大类已有 5 页：编辑距离（min 代价）、0-1 背包（max 价值）、LCS（填表 + 回溯）、LIS（一维）、完全背包（可重复取 max）。这些 DP 求的都是「最优值 / 解」。还差一味常见的——**计数 DP**：不求最优，而是**数方案数**。

**硬币找零方案数**（LeetCode 518，Coin Change II）：给若干面额、每种无限枚，问凑出目标金额有**多少种**不同组合。它和完全背包同为「无限次取」的二维表，但递推是**相加（累计方案数）**而非取 max，边界也从 0 变成 `dp[0][0]=1`（凑 0 元有 1 种方案：什么都不选）。这正好补齐 DP 的「计数」维度。

## 目标

在动态规划大类新增第 6 页「硬币找零方案数」，接入算法播放器（`AlgorithmPlayer`）：

1. **纯复用 MatrixView 矩阵轨**（第 7 消费者，零改动）：行=硬币面额（∅/1/2/5）、列=金额（0..5），逐格填「方案数」DP 表，`active` 琥珀环、`sources` 黄高亮来源格、`updatedCell` 绿闪。
2. **与完全背包对照**：同为无限次取的二维表、「用一枚」的来源也在**本行** `dp[i][a-面额]`；差别只在——完全背包取 `max(不取, 取)`，硬币找零取 `dp[i-1][a] + dp[i][a-面额]`（**方案数相加**）；边界 `dp[0][0]=1`。
3. 固定输入：硬币 `[1,2,5]`、金额 `5` → **4 种**方案（`1×5`、`1×3+2`、`1+2×2`、`5`）；DP 表右下角 `dp[3][5]=4`。

## 非目标

- 不做「凑出金额的最少硬币数」（Coin Change I，那是 min DP，与背包同族）——本页聚焦「方案数」。
- 不做一维滚动数组优化版（二维表利于与完全背包对照）。
- 不改 MatrixView、不改 AlgorithmPlayer。
