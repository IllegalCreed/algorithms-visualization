## 背景

「回溯与搜索」大类已有 N 皇后（[[C-20260703-055]]，棋盘约束）、子集生成（[[C-20260703-056]]，选/不选二叉决策树）、全排列（[[C-20260703-057]]，多叉决策树，复用决策树轨）。三页把回溯的「试探 + 回溯」讲清了，但**剪枝**——回溯高效的关键——还没有一页专门演示。

本变更做**组合总和（Combination Sum）**：从一组数里选若干个（每个最多用一次）凑出目标和。它天然带**剪枝**：一旦当前和**超过目标**，这条分支不可能成解，直接砍掉、不再展开。这补齐回溯「试探-剪枝-回溯」三要素里最缺的一环，也是 DecisionTreeView 的第 3 个消费者（首次需要「剪枝支」表达）。

## 目标

1. 新增**组合总和**算法页（`/docs/combination-sum`），接入既有 AlgorithmPlayer 框架（正文 + 分步播放器 + 四语言代码同步高亮）。
2. **复用并小幅扩展 DecisionTreeView**：给决策树轨加 `prunedIds`（剪枝节点），View 加 `.pruned`（红/虚线剪枝支）——从全排列的「零改动复用」升级为「按新需求小扩展」通用原语。
3. 逐步呈现组合总和的决策树 DFS 与剪枝：**加一个数 → 和 ≤ 目标就继续 / 和 > 目标就剪枝 / 和 = 目标就记录 → 回溯换下一个数**。
4. 在「回溯与搜索」大类下新增第 4 个条目（菜单 + 首页网格）。

## 用户故事

- 作为学习者，我想**亲眼看到剪枝**：某些分支因为「加上去会超过目标」而被红叉砍掉、不再展开，理解剪枝为什么能大幅减少搜索。
- 作为学习者，我想看到「当前和」随决策增长、与目标比较，建立「和 > 目标 → 剪枝、和 = 目标 → 记录」的判断。
- 作为学习者，我想对照 N 皇后 / 子集 / 排列 / 组合总和，把回溯「试探 + 剪枝 + 回溯」三要素补全。
