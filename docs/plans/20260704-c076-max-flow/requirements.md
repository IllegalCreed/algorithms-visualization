## 背景

图算法大类已铺开最短路（Dijkstra/Bellman-Ford/Floyd）、MST（Kruskal/Prim）、拓扑排序、强连通分量、2-SAT 共 8 页。本页补图论的皇冠——**网络流 / 最大流**。

**最大流问题**：有向图每条边有**容量**，从**源 s** 到**汇 t**最多能送多少流量（每条边流量 ≤ 容量，每个中间点流入=流出）？**Ford-Fulkerson 方法**：反复在**残量网络**里找一条从 s 到 t 的**增广路**、沿路把流推满瓶颈，直到再也找不到增广路。残量网络的关键是**反向边**——每条已用了流的边 `u→v` 会产生一条容量为「已用流量」的反向边 `v→u`，允许后续**把之前的流退回、改道**（贪心走错也能反悔）。终止时的流就是最大流，且 **最大流 = 最小割**（把 s、t 分开、割断的边的容量之和最小）——图论最深刻的对偶定理之一。用 BFS 找最短增广路即 **Edmonds-Karp**（O(VE²)）。

## 目标

在图算法大类新增第 9 页「最大流」，接入算法播放器（`AlgorithmPlayer`），**复用 GraphView，不新建轨**：

1. **网络展示**：有向图每条边标「流量/容量」（初始 `0/cap`），源 s、汇 t 标记。
2. **逐轮增广**：每轮先**找增广路**（高亮路径 + 瓶颈值），再**沿路增流**（更新流量标签）。故意让第一条增广路走「贪心陷阱」（经过中间边 `a→b`）。
3. **反向边反悔**：某轮增广路**反向经过**一条已用流的边（红色高亮），把之前误走的流**退回、改道**——展示残量网络的精髓。
4. **最大流 = 最小割**：走到无增广路，给出最大流值，并高亮**最小割**（把 s 侧与 t 侧分开的饱和边）。
5. 固定网络 `s→a:3, s→b:3, a→b:1, a→t:3, b→t:3` → 4 轮增广（`s→a→b→t`(1)、`s→a→t`(2)、`s→b→t`(2)、`s→b→a→t`(1，反向边 `b→a`)）；**最大流 6 = 最小割 {s}|{a,b,t}（`s→a`+`s→b`=6）**。

## 非目标

- 不做 Dinic / ISAP / 预流推进等高级最大流算法——本页讲清 Ford-Fulkerson 方法 + 残量网络思想，prose 点到 Edmonds-Karp。
- 不做最小费用最大流 / 多源多汇——固定单源单汇整数容量小网络。
- 不单独画残量反向边（避免图面翻倍）：反向增广时高亮原边 + 流量标签减少来表达「退流」。
- 不改 AlgorithmPlayer；GraphView 仅 additive 加 `edgeLabel`（流量/容量标签）+ `.reverse` 边样式（其它算法零回归）。
