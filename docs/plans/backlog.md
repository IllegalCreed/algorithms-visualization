## 深度项细节（现有页对应的「当年所砍」）

各页 requirements 的「不做什么」即本池深度项来源，便于复用当时的论证：

- **树**（C-019）：弃了**平衡树 AVL/红黑**、删除节点、n 叉树、序列化 → D1（+ 删除可顺带）。
- **哈希**（C-021）：弃了**开放寻址**、**rehash 扩容**、真实字符串散列、删除 → D2。
- **链表**（C-018）：弃了**双向链表**、循环链表、跳表 → D3（跳表见 B4）。
- **队列**（C-016）：弃了**循环队列 / 双端队列 / 优先队列** → D4。
- **数组**（C-017）：弃了**动态扩容**、多维、稀疏 → D5。
- **堆**（C-020）：弃了小顶堆（镜像）、heapify 建堆、d 叉堆 → 可选小补（优先级低）。
- **图**（C-022）：弃了有向 / 带权 / Dijkstra / 拓扑 / 最小生成树 / 邻接矩阵互动 → **归「图算法」独立轨道，不在本池**（见下）。

## 不在本池（另列）

- **图算法线**：有向图 / 带权图 / 最短路（Dijkstra·Bellman-Ford）/ 拓扑排序 / 最小生成树（Prim·Kruskal）/ 连通分量——这些更像「算法」而非「结构」，建议单独里程碑/菜单分类，不混进数据结构。
- **通用化重构**：把树/图二维布局抽成可复用插槽组件（工程优化，非内容）。

## 落点决策须知（启动每项前定）

- **现有页加节（D1–D5 多数）**：只改对应 `*.vue` 正文 +（可能）新互动子组件；**不动菜单/路由/首页**。注意正文变长是否拆姊妹页。
- **新页（B1–B7）**：需新增 侧边菜单条目（`Docs/Menu/hooks.ts`）+ 路由（`router/index.ts`）+ 首页网格条目 + 图标（`assets/*.svg`）+ `Article/DataStructure/Xxx.vue`，与 M3 各结构同范式。
- Case ID 命名空间：复用页加节沿用原页命名；新结构用新前缀（如 `TC-TRIE-*` / `TC-UNIONFIND-*`），避让既有。
