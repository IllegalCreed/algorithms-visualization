## 背景

M4 广度第七项 B7（进阶），**M4 广度收官之作**。新增**布隆过滤器 Bloom Filter** 独立页。backlog 点名「海量去重 / 存在性（概率型）」。

**定位**：布隆过滤器是**概率型「存在性」数据结构**——只回答「x **可能在**集合里 / x **一定不在**」。它用一个**位数组**（m 位）+ **k 个哈希函数**：

- **加入 x**：用 k 个哈希算出 k 个位置，把这些位都置 1。
- **查询 x**：算出 k 个位置，**全为 1** → 可能存在（也许误判）；**有任一为 0** → 一定不存在。

核心特性：**会误判（false positive，假阳性），但绝不漏判（no false negative）**——「说不在就一定不在，说在可能是误判」。换来的是**极省空间 + O(k) 常数时间**。标准版不支持删除。用于缓存穿透防护、爬虫 URL 去重、海量黑名单/白名单预筛。

**新页 vs 加节**：广度项需 **4 处接线**——路由 + 侧边菜单 + 首页网格 + 图标（套路已跑通 6 次），外加 `Article/DataStructure/BloomFilter.vue`。

## 三个地基决策

1. **固定结构**（仿前面所有进阶结构定死→可单测）：位数组 **m = 16 位**（bit 0..15），**k = 3** 个**确定性哈希**：`h1(x)=x%16`、`h2(x)=(7x)%16`、`h3(x)=(11x+5)%16`。三者对小整数较分散，且可手算验证。
2. **演「位数组 + 多哈希」的加入/查询**：加入把 3 个哈希位置 1；查询检查 3 位是否全 1。可视化点亮被探测的位 + 高亮已置 1 的位，把「全 1=可能存在 / 有 0=一定不存在」一眼呈现。
3. **演「误判」（教学杀手锏）**：`useBloom` 额外**偷偷记录真实加入集合**（仅用于教学标注，不参与布隆判断）。加入 3、7、11 后，查询从没加过的 **2**（哈希位 `[2,14,11]` 恰好都被覆盖）→ 布隆说「可能存在」，但比对真实集合发现没加过 → 点破「这是假阳性误判」。这是布隆过滤器最精髓、最反直觉的一课。

## 与现有结构页的关系（一句话）

`BloomViz` 复用「一排格子 + 探测点亮 + 状态解说」画法（近似哈希表 HashViz 的桶阵列），但格子是**位**（0/1）、查询走**多哈希取交**、并独家演示「**误判**」——讲哈希表都没讲的「概率型、省空间、可假阳性」。
