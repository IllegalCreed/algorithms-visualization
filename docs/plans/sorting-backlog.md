## 背景

M3 已完成 **8 个基础排序**：冒泡 / 选择 / 插入 / 希尔 / 归并 / 快速 / 堆排 / 计数。盘点后发现——按「排序思想族」分，**每族基本只有 1 个代表**，大量细分变体、线性排序、工程级混合排序**没有照顾到**（2026-06-30 Owner 指出）：

| 思想族   | 已有         | 缺的细分                                             |
| -------- | ------------ | ---------------------------------------------------- |
| 交换类   | 冒泡、快速   | 三路快排、双轴快排、鸡尾酒（双向冒泡）               |

| 插入类   | 插入、希尔   | 二分插入                                             |

三块最明显：① **线性排序没补齐**（基数 / 桶）；② **主流排序的工程变体几乎没有**（三路 / 双轴快排、自底向上归并）；③ **真实世界用的混合排序完全空白**（TimSort / IntroSort）。

本池负责规划这条「排序细分扩充」线（roadmap M7，产品内容线，与 M5 增长线、M6 新算法大类扩充并行、互不冲突）。排序属**已存在的大类**，故是「在已有排序大类里补细分」（类似 M4 数据结构的深化扩充），不新增顶层分类。

## 各项细节（★ = 阶段内建议先做）

### 阶段一 · 线性排序补齐

- **S1 ★ 基数排序 Radix Sort**（LSD）：按个位 → 十位 → 百位，每轮把元素分配到 0–9 共 10 个桶、再按桶序收集；多轮后整体有序。可视化：当前处理位高亮 + 10 桶分配动画 + 收集回主轨（复用计数排序的 `CountView` 桶轨思路）。线性 O(d·(n+k))。
- **S2 桶排序 Bucket Sort**：按值域均分若干桶，元素入对应桶，桶内各自排序（插入），再按桶序拼接。可视化：分桶 + 桶内排序 + 合并。和计数 / 基数同属非比较线性排序，三件套补齐。

### 阶段二 · 主流排序的工程变体

- **S3 ★ 三路快排 3-way Quicksort**：荷兰国旗划分，把数组分成「< pivot / = pivot / > pivot」三段，等于 pivot 的一次性归位——大量重复元素时远胜普通快排。可视化：三色三段 + lt/gt/i 三指针。复用快排的 `BarsView`。
- **S4 双轴快排 Dual-Pivot Quicksort**：取两个 pivot（p1≤p2）把数组分三段（<p1 / [p1,p2] / >p2），Java 基本类型 `Arrays.sort` 用的就是它。可视化：双 pivot + 三段 + 多指针。
- **S5 自底向上归并 Bottom-up Merge Sort**：不递归，子数组宽度从 1 开始，逐轮两两合并（1→2→4→8…）。和已有递归归并对照「同一算法两种写法」。可视化：每轮按宽度成对合并区间高亮。
- **S6 二分插入排序 Binary Insertion Sort**：插入排序里用折半查找定位插入点（比较从 O(n) 降到 O(log n)，移动仍 O(n)）。对照普通插入。可视化：折半查找区间收缩 + 插入。
- **S7 鸡尾酒排序 Cocktail Shaker Sort**：双向冒泡，一趟从左到右冒最大、再从右到左冒最小，来回缩小范围。冒泡的小优化、直观。

### 阶段三 · 工程级混合排序（进阶硬核）

- **S8 TimSort**：识别已有序的「run」（升/降段）、小段用二分插入补到 minrun、再用归并按规则合并（含 galloping）。Python `sorted` / Java 对象 `Arrays.sort` 用的就是它。可视化：run 识别 + 小段插入 + 合并栈。范式较新，可分步呈现。
- **S9 IntroSort**：以快排为主，递归深度超过 `2·log n` 时转堆排（防最坏 O(n²)），小段（<16）转插入。C++ STL `std::sort` 实现。可视化：快排为主 + 深度计数 + 触发切换时的策略转换。

## 不在本池（趣味 / 价值低，按需另议）

- 奇偶排序 Odd-even、地精排序 Gnome、煎饼排序 Pancake、bogosort、sleep sort 等趣味排序。
- 外部排序（多路归并 + 磁盘）、并行排序——偏工程系统，另议。
